## Get started with ChartDB

ChartDB is a web-based database diagram editor that visualizes your database schema with a single query. No installations, no database passwords required.

<Info>
  This guide uses the **cloud version** at [app.chartdb.io](https://app.chartdb.io). For self-hosting options, see the [Installation](/installation) guide.
</Info>

## Create your first diagram

<Steps>
  <Step title="Open ChartDB">
    Navigate to [app.chartdb.io](https://app.chartdb.io) in your browser. No account or login required.
  </Step>

  <Step title="Choose your database type">
    Select your database from the list:
    - PostgreSQL (including Supabase, TimescaleDB)
    - MySQL
    - SQL Server
    - MariaDB
    - SQLite (including Cloudflare D1)
    - ClickHouse
    - CockroachDB
    - Oracle
  </Step>

  <Step title="Run the Smart Query">
    ChartDB provides a database-specific query that extracts your schema as JSON metadata. Copy the query and run it in your database client.

    <Tabs>
      <Tab title="PostgreSQL">
        ```sql
        SELECT json_build_object(
          'tables', (
            SELECT json_agg(json_build_object(
              'schema', schemaname,
              'table', tablename
            ))
            FROM pg_tables
            WHERE schemaname NOT IN ('pg_catalog', 'information_schema')
          ),
          'columns', (
            SELECT json_agg(json_build_object(
              'schema', table_schema,
              'table', table_name,
              'column', column_name,
              'type', data_type,
              'nullable', is_nullable = 'YES',
              'default', column_default
            ))
            FROM information_schema.columns
            WHERE table_schema NOT IN ('pg_catalog', 'information_schema')
          ),
          'pk_info', (
            SELECT json_agg(json_build_object(
              'schema', tc.table_schema,
              'table', tc.table_name,
              'constraint_name', tc.constraint_name,
              'column', kcu.column_name
            ))
            FROM information_schema.table_constraints tc
            JOIN information_schema.key_column_usage kcu
              ON tc.constraint_name = kcu.constraint_name
              AND tc.table_schema = kcu.table_schema
            WHERE tc.constraint_type = 'PRIMARY KEY'
              AND tc.table_schema NOT IN ('pg_catalog', 'information_schema')
          ),
          'fk_info', (
            SELECT json_agg(json_build_object(
              'schema', tc.table_schema,
              'table', tc.table_name,
              'constraint_name', tc.constraint_name,
              'column', kcu.column_name,
              'reference_schema', ccu.table_schema,
              'reference_table', ccu.table_name,
              'reference_column', ccu.column_name
            ))
            FROM information_schema.table_constraints tc
            JOIN information_schema.key_column_usage kcu
              ON tc.constraint_name = kcu.constraint_name
              AND tc.table_schema = kcu.table_schema
            JOIN information_schema.constraint_column_usage ccu
              ON tc.constraint_name = ccu.constraint_name
              AND tc.table_schema = ccu.table_schema
            WHERE tc.constraint_type = 'FOREIGN KEY'
              AND tc.table_schema NOT IN ('pg_catalog', 'information_schema')
          ),
          'indexes', (
            SELECT json_agg(json_build_object(
              'schema', schemaname,
              'table', tablename,
              'index_name', indexname,
              'column', indexdef
            ))
            FROM pg_indexes
            WHERE schemaname NOT IN ('pg_catalog', 'information_schema')
          )
        ) as metadata;
        ```
      </Tab>

      <Tab title="MySQL">
        ```sql
        SELECT JSON_OBJECT(
          'tables', (
            SELECT JSON_ARRAYAGG(JSON_OBJECT(
              'schema', TABLE_SCHEMA,
              'table', TABLE_NAME
            ))
            FROM information_schema.TABLES
            WHERE TABLE_SCHEMA NOT IN ('information_schema', 'mysql', 'performance_schema', 'sys')
          ),
          'columns', (
            SELECT JSON_ARRAYAGG(JSON_OBJECT(
              'schema', TABLE_SCHEMA,
              'table', TABLE_NAME,
              'column', COLUMN_NAME,
              'type', DATA_TYPE,
              'nullable', IS_NULLABLE = 'YES',
              'default', COLUMN_DEFAULT,
              'extra', EXTRA
            ))
            FROM information_schema.COLUMNS
            WHERE TABLE_SCHEMA NOT IN ('information_schema', 'mysql', 'performance_schema', 'sys')
          )
        ) as metadata;
        ```
      </Tab>

      <Tab title="SQLite">
        For SQLite, use the file upload method or manually run:
        
        ```sql
        SELECT name, sql FROM sqlite_master WHERE type='table';
        ```
        
        Then format the output as JSON or use the DDL import method instead.
      </Tab>
    </Tabs>

    <Note>
      The Smart Query extracts metadata only—no actual data. Your database credentials are never sent to ChartDB servers.
    </Note>
  </Step>

  <Step title="Paste the JSON result">
    Copy the entire JSON output from your query results and paste it into the ChartDB import dialog.
    
    ChartDB will automatically:
    - Create tables with all columns
    - Draw relationships based on foreign keys
    - Set up indexes and constraints
    - Organize tables on the canvas
  </Step>

  <Step title="Explore and edit">
    Your database schema is now visualized! You can:
    - **Drag tables** to rearrange the layout
    - **Edit tables** by clicking on them
    - **Add relationships** by dragging between tables
    - **Create areas** to group related tables
    - **Add notes** for documentation
    - **Export** to SQL DDL, DBML, or PNG image
  </Step>
</Steps>

## Alternative import methods

If the Smart Query doesn't work for your database, try these alternatives:

<Tabs>
  <Tab title="DDL Scripts">
    Import from SQL DDL scripts:
    
    ```sql
    CREATE TABLE users (
      id SERIAL PRIMARY KEY,
      email VARCHAR(255) UNIQUE NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    CREATE TABLE orders (
      id SERIAL PRIMARY KEY,
      user_id INTEGER REFERENCES users(id),
      total DECIMAL(10, 2),
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    ```
    
    [Learn more about DDL import](/features/ddl-import)
  </Tab>

  <Tab title="DBML">
    Import from DBML (Database Markup Language):
    
    ```dbml
    Table users {
      id integer [primary key]
      email varchar(255) [unique, not null]
      created_at timestamp [default: `now()`]
    }

    Table orders {
      id integer [primary key]
      user_id integer [ref: > users.id]
      total decimal(10,2)
      created_at timestamp [default: `now()`]
    }
    ```
    
    [Learn more about DBML import](/features/dbml-import)
  </Tab>

  <Tab title="Start from Scratch">
    Create a new diagram and add tables manually:
    
    1. Click **New Diagram**
    2. Select your database type
    3. Click **Add Table** on the canvas
    4. Define fields, indexes, and constraints
    5. Draw relationships between tables
    
    [Learn more about creating diagrams](/guides/creating-diagrams)
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Query returns error or permission denied">
    Some databases require specific permissions to query metadata views. Try:
    
    - **PostgreSQL**: Grant access to `information_schema` and `pg_catalog`
    - **MySQL**: Ensure user has `SELECT` privilege on `information_schema`
    - **SQL Server**: Requires `VIEW DEFINITION` permission
    
    If permissions are restricted, use the **DDL import** method instead.
  </Accordion>

  <Accordion title="JSON output is too large">
    For databases with many tables (500+), the JSON metadata can be very large. Try:
    
    - Filter by schema: Modify the query to exclude schemas you don't need
    - Import specific tables: Use the table selection feature in ChartDB
    - Use DDL import: Selectively import only the tables you want to visualize
  </Accordion>

  <Accordion title="Relationships are missing">
    If foreign key relationships don't appear:
    
    - Verify foreign keys exist in your database (run `\d table_name` in PostgreSQL)
    - Check that the Smart Query includes the `fk_info` section
    - For SQLite: Foreign keys must be enabled with `PRAGMA foreign_keys = ON`
    - Manually create relationships in ChartDB if needed
  </Accordion>

  <Accordion title="Can't see my tables">
    If tables don't appear after import:
    
    - Check the query filtered out system schemas (pg_catalog, information_schema, etc.)
    - Verify tables exist in your database: `SELECT * FROM information_schema.tables`
    - Try the **DDL import** method as an alternative
    - Ensure JSON output is valid and complete
  </Accordion>
</AccordionGroup>
